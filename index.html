Parallel Wave Function Collapse

The wave function collapse algorithm simulates quantum systems

by modeling the behavior of wave functions. In quantum

mechanics, a wave function represents a state of a quantum

system by encompassing all possible finite states at once, known

as ‘superposition’. Only when an observation is made, does a wave

function collapse and converge to a single definite state.

Goal

The wave function collapse algorithm is renowned for its ability to simulate quantum systems, but its computational intensity can lead to input size limitations and long delays. Our team aims to tackle these challenges by introducing parallelism to tasks that can be run

Image Processor

The image processor component is the glue that allows us to prepare the image to be used by the wave function collapse core algorithm. The purpose of the image processor is for the generation of useful information like the samples and their ids, and how they associate with the adjacency and frequency requirements.

There are three main parts to consider in this segment.

  1. The NxN permutation of the samples, with wrapping in mind.
  1. The frequency rule generation.
  1. The adjacency rule generation.

Sample Permutation

Permuting the samples is done by stepping over every possible pixel and grabbing all N-1 pixels east-ward and south-ward, from this we get the raw samples.

Frequency Rule

Using the raw samples can be troublesome, it is required that we deduplicate them. Conveniently, this process kills two birds with one stone, we could also count the number of duplicates and use the count in our frequency rule.

// src/model.rs
// Calculate the number of times each unique sample appears
let freq_map: HashMap<Sample, i32> = unprocessed_samples.iter().fold(
		HashMap::<Sample, i32>::new(),
		|mut freq_map, sample| {
				*freq_map.entry(sample.clone()).or_insert(0) += 1;
				freq_map
		},
);

Adjacency Rule

The last piece of the puzzle is generating the adjacency rules. This is achieved by checking every sample against each other (itself included) and seeing if their N-1 pixels in their respective direction matches. (E.g. The bottom N-1 pixels of sample A matches the top N-1 pixels of sample B) 

// src/model.rs
// Create adjacency rules
for s1 in 0..samples.len() {
    for s2 in 0..samples.len() {
        for direction in &ALL_DIRECTIONS {
             if samples[s1].compatible(&samples[s2], *direction) {
                adjacency_rules[s1][direction.to_idx()].insert(s2);
             }
        }
    }
}
